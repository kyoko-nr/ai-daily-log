# Change: Supabase 認証フローの追加（ログイン/サインアップ/ログアウト/リセット）

## Why
ユーザーが日次ログとサマリーに安全にアクセスできる必要があります。
現状は認証フローが存在しないため、アクセス制御を実現できません。

## What Changes
- Supabase Auth によるメール/パスワードのログイン、サインアップ、
  ログアウト、パスワードリセット要求を追加する。
- `/login` にログイン、サインアップ、リセット要求の UI を追加する。
- `/app` をサーバーサイド認証で保護する（middleware には依存しない）。
- 認証済みは `/app`、未認証は `/login` へリダイレクトする。
- `/app` にログアウト操作を追加する。

## Impact
- 影響する仕様: `specs/authenticate-user/spec.md` (新規)
- 影響するコード: `src/app`, `src/lib/supabase`, サーバーサイド認証ハンドラ
