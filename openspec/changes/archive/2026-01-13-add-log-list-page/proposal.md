# Change: 過去ログ一覧ページ（/logs）の追加

## Why
保存した日次ログを月単位で見返せないため、振り返りやサマリー作成の前提となる
「過去ログの閲覧」がしづらい。

## What Changes
- 認証済みユーザー向けに `/logs` の一覧ページを追加する。
- 初期表示では「現在の月」のログを降順（新しい日付が先）で表示する。
- 前月/次月へ移動するボタンを提供し、選択中の年月のログを取得して再表示する。
- 選択中の年月は URL パラメータではなく atom で内部的に管理する。
- 一覧の各ログはリンクとし、押下で詳細ページへ遷移できるようにする
  （詳細ページの内容は TODO とし、最小のプレースホルダー表示のみ行う）。

## Assumptions
- 月の選択状態は `/logs` 内で atom として保持し、ページ再読み込み時は現在の月に戻る。
- ログは `daily_logs`（`log_date`, `log_text`）を取得し、一覧には `log_date` と
  `log_text` の短い抜粋を表示する。

## Non-goals
- 月指定の deep link（例: `/logs?month=...`）の提供。
- 詳細ページでの本文/質問/回答の表示（本 change では TODO）。
- ページネーションや全文検索、タグ付け等の高度な絞り込み。

## Impact
- 影響する仕様: `specs/daily-log-listing/spec.md`（新規）
- 影響するコード（想定）: `src/app/logs`, `src/app/api/logs`, `src/lib/supabase`
