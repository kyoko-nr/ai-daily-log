# Change: ログ詳細ページ（`/logs/[logId]`）の実装

## Why
過去ログ一覧（`/logs`）から詳細ページへ遷移できる導線はあるが、現状はプレースホルダー表示のみで
ログ本文やフォローアップ質問/回答を見返せない。振り返りや後続の「修正/削除」導入の前提として、
読み取り専用の詳細表示が必要。

## What Changes
- 認証済みユーザー向けに `GET /logs/[logId]` の詳細ページを実装する。
- 詳細ページでは日付、ログ本文、フォローアップ質問/回答を **入力欄ではなく表示（label + text）** として提供する。
- 詳細セクション右上に「削除」「修正」ボタンを配置する（本 change ではハンドラ/遷移は TODO として未実装）。
- 存在しない `logId`（または参照権限がない）場合は `/logs` へフォールバックする。

## Assumptions
- 参照データは `daily_logs`（`log_date`, `log_text`）と `daily_log_followups`（複数行）から取得する。
- フォローアップの表示順は `created_at` 昇順（同値は `id`）とする。
- 未回答（`answer = ''`）は空表示とし、未回答を示す文言は表示しない。
- フォローアップが 0 件の場合、フォローアップ領域自体を表示しない（「ない旨」のメッセージは出さない）。
- 「削除」「修正」ボタンは UI のみ配置し、押下時の挙動は実装しない（disabled もしくは TODO 表示）。

## Decisions
- ボタンは詳細セクションの右上に配置する。
- 未回答は空表示とする。
- ルートは現状どおり `logs/[logId]`（`logId`）のままとする。

## Non-goals
- ログの削除・修正の実装（API、確認ダイアログ、遷移、編集フォーム等）。
- 共有リンク、検索、ページネーション等の閲覧機能拡張。
- 既存のログ保存・一覧取得の仕様変更。

## Impact
- 影響する仕様（差分）:
  - `daily-log-detail`（新規）
  - `daily-log-listing`（詳細ページ TODO の解消と参照先の明確化）
- 影響するコード（想定）:
  - `src/app/logs/[logId]/page.tsx`
  - `src/app/logs/hooks/*`（詳細取得ユーティリティ追加の可能性）
  - `src/lib/schemas/dailyLogSchemas.ts`（詳細表示用スキーマ追加の可能性）
